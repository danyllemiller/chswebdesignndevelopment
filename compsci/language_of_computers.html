5.1 Bits, Bytes, and Binary

If you rip open a computer, you won't find a tiny dictionary or a color palette inside. You will only find electricity. Because computers are electrical machines, they only understand two absolute states: Power ON and Power OFF.

5.1.1 The Binary Switch

Humans count using Base-10 (Decimal), meaning we use ten distinct digits (0 through 9). Why? Probably because we have ten fingers! Computers, however, use Base-2 (Binary), which uses only two digits: 1 and 0.

Inside the CPU, there are billions of microscopic Transistors. You can think of a transistor as a tiny light switch. If the switch is turned ON and electricity is flowing, the computer registers a 1. If the switch is OFF and there is no electricity, it registers a 0. By combining billions of these tiny switches, a computer can represent incredibly complex data.

5.1.2 Counting in Binary

In our human Base-10 system, each place value is multiplied by 10 (the Ones place, the Tens place, the Hundreds place). In Base-2, each place value is multiplied by 2 ($2^0, 2^1, 2^2, 2^3, \dots$). This gives us the binary place values of 1, 2, 4, 8, 16, 32, 64, and 128.

Step-by-Step: How to Convert Binary to Decimal

Let's translate the 8-bit binary number 01001001 into a human number using the "Flipper" method.

1286432168421
01001001
  1. Draw the Table: Write out the powers of two from right to left (1, 2, 4, 8...).
  2. Drop in the Code: Write your binary number underneath the columns.
  3. Add the "ON" Switches: Anywhere there is a 1, take the number at the top of that column. Ignore the columns with a 0.
  4. Calculate: Add them together. (64 + 8 + 1) = 53. The binary code 01001001 is the number 53!

5.1.3 Measuring Data (Bits to Terabytes)

A single 1 or 0 is called a Bit (Binary Digit). Eight bits put together is called a Byte. From there, data scales up incredibly fast, but unlike human math, computer math scales by multiples of 1,024 (because $2^{10} = 1024$).

Kilobyte (KB)

1,024 Bytes.
About 1 page of text.

Megabyte (MB)

1,024 KB.
A high-res photo or MP3.

Gigabyte (GB)

1,024 MB.
A 1-hour HD movie.

Terabyte (TB)

1,024 GB.
A modern computer's SSD.

5.2 Translating the World Standard 9-12.DA.S.1

We know how computers store numbers. But how do you save a text message, a photograph, or the color of a button on a website when the computer only speaks math? We need a translation key.

5.2.1 Representing Text (ASCII & Unicode)

To store letters, computer scientists assigned a specific binary number to every letter of the alphabet. The original standard was called ASCII (American Standard Code for Information Interchange). In ASCII, the capital letter 'A' is assigned the number 65 (01000001 in binary). A lowercase 'a' is 95.

The problem? ASCII only used 5 bits, meaning it could only hold 128 characters. This was fine for English, but terrible for the rest of the world. Today, we use Unicode. Unicode uses up to 32 bits, allowing for millions of characters. This upgrade allowed computers to render Japanese Kanji, Arabic script, and of course, Emojis (😎 is Unicode number U+1F60E).

5.2.2 The Language of Color (Hexadecimal)

If you want to tell a computer screen to display a specific color, you use the RGB (Red, Green, Blue) model. By mixing different intensities of Red, Green, and Blue light, you can create millions of colors.

Hexadecimal (Base-16) & The CTE Connection

Instead of writing out massive binary strings for colors, Web Designers use Hexadecimal (Hex). Hex is a Base-16 system. It uses the numbers 0-9, and then the letters A-F to represent 10-15.

A Hex color code looks like this: #FF0000. It is broken into three pairs: Red (FF), Green (00), and Blue (00). Because F is the maximum value, #FF0000 tells the screen to turn the Red light up to 100%, and keep Green and Blue turned completely off. The result is a bright, vibrant red. This is exactly how you color elements using CSS in Web Design!

Step-by-Step: How to Find a Hex Color Code

If you see a beautiful color on a website and want to use it in your own code, you can use Google Chrome's built-in developer tools to steal the Hex Code.

  1. Inspect the Page: Right-click anywhere on the webpage and select "Inspect" (or press Ctrl+Shift+I / Cmd+Option+I).
  2. Find the CSS: In the panel that opens, look for the "Styles" tab at the bottom right.
  3. Activate the Eyedropper: Look for a small square color box next to any CSS rule (like background-color: #ffffff;). Click the colored square.
  4. Sample the Color: A color picker will open. Click the Eyedropper Icon, then move your mouse over the actual webpage. Click on the color you want to steal, and the exact Hex Code (e.g., #2A9D8F) will be revealed!

5.2.3 Digital Imagery (Pixels & Bit Depth)

A digital photograph is just a massive grid of tiny, square lights called Pixels (Picture Elements). The computer stores a Hex color code for every single pixel in the grid.

The quality of an image depends on its Bit Depth. An 8-bit image can only display 256 colors (making it look like a retro Super Nintendo game). A modern 24-bit "True Color" image can display 16.5 million colors, creating a photorealistic picture.

5.3 Data Abstraction & Compression

If a 4K movie stores 24-bit color data for over 8 million pixels, 60 times a second, the file size would be too massive to download. To fix this, computer scientists use Compression algorithms to shrink the data.

5.3.1 Lossy vs. Lossless Compression

There are two main ways to make a file smaller: throw data away, or organize it better using math.

Lossy Compression (Throwing Data Away)

Lossy algorithms permanently delete data that human eyes or ears likely won't notice. For example, if a photo has 100 slightly different shades of blue in the sky, a Lossy algorithm will delete 90 of them and make the sky one uniform blue. This shrinks the file size massively, but you permanently lose visual quality.

Common formats: .JPG (Images), .MP3 (Audio), .MP4 (Video)

Lossless Compression (Mathematical Packing)

Lossless algorithms shrink the file size without deleting a single piece of original data. It does this by finding patterns. Instead of saving "Blue, Blue, Blue, Blue," it saves the math equation "4x Blue." When the file is opened, it is reconstructed perfectly to its exact original state.

Common formats: .PNG (Images), .ZIP (Folders), .FLAC (Audio)

5.3.2 Sound and Video as Data

Real-world sound is an analog, continuous wave. A computer can't store a continuous wave. Instead, it takes thousands of tiny digital "snapshots" of the wave every second. This is called the Sampling Rate. CD-quality audio is sampled 44,100 times per second! The higher the sampling rate, the closer the digital file sounds to the real, original audio.

5.3.3 The Tradeoffs of Representation

In computer science, you must always choose the right format for the job. A professional photographer needs a massive, uncompressed RAW or TIFF file so they have all the data available for editing in Photoshop. A Web Designer, however, needs small, highly compressed JPGs or WebP files so their website loads quickly on mobile phones. Everything is a tradeoff between quality and performance.
