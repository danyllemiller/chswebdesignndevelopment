10.1 Conditional Statements Standard 9-12.AP.C.1

A program that always runs in a straight line is rare. Most programs need to make decisions based on changing conditions. This is where we use Conditional Statements.

10.1.1 The If Statement

The most basic control structure is the If Statement. It checks a condition and executes code only when that condition is true.

If-Then Logic
  • IF (Condition) — Is the condition true?
  • THEN (Action) — Execute this code.
  • ELSE (Alternative) — Do this instead if the condition is false.

10.1.2 Boolean Logic

Behind every condition is a Boolean value—either true or false. Computers make decisions by comparing values using these operators:

==

Equal to

!=

Not equal to

>

Greater than

<

Less than

10.1.3 Compound Conditions (AND / OR)

Real-world decisions often require checking multiple conditions at once. We use logical operators to combine them:

Combining Conditions
AND

BOTH conditions must be true.
"If age >= 13 AND have_permission, then allow account creation."

OR

AT LEAST ONE condition must be true.
"If ticket == 'student' OR ticket == 'teacher', then give discount."

10.2 Loops & Iteration Standard 9-12.AP.C.2

One of the most powerful features of computers is their ability to repeat tasks infinitely without getting tired. Loops allow us to execute the same code multiple times.

10.2.1 The For Loop

A For Loop is used when you know exactly how many times you want to repeat an action.

For Loop Structure

FOR (counter) FROM 1 TO 10:DO the action

This runs exactly 10 times!

10.2.2 The While Loop

A While Loop is used when you don't know how many times to repeat, but you know the condition that should stop the loop.

Pre-Test Loop

Checks the condition before running. If false, never runs at all.

WHILE (battery > 0) { playGame() }

Post-Test Loop

Runs at least once, then checks condition.

DO { playGame() } WHILE (battery > 0)

10.2.3 Avoiding Infinite Loops

The most dangerous bug is an Infinite Loop—a loop that never stops. This happens when the condition to stop the loop is never reached.

10.3 Event-Driven Programming

In interactive apps, programs don't run in a predictable sequence. Instead, they wait for things to happen—like a user clicking a button. This is Event-Driven Programming.

10.3.1 Understanding Events

An Event is an action that happens while a program is running. The program "listens" for these events and responds when they occur.

Click

User clicks a button

Keypress

User presses a key

Timer

A specific time passes

10.3.2 Event Handlers

An Event Handler is code that runs when a specific event occurs. It's like having a security guard who only acts when someone rings the doorbell.

Event Handler Pattern

Most programming languages use this pattern:

  1. Identify the Event: What happened? (e.g., "button_click")
  2. Connect to Handler: "When button_click occurs, run this function."
  3. The Handler Function: The specific code that executes.

10.3.3 The Event Loop

Modern apps run inside an Event Loop—a continuously running program that checks for events, handles them, and then waits for the next one. This is how video games, apps, and interactive websites work!
