Images (The Right Way)

Images are the heavy lifters of web design. They evoke emotion, clarify complex topics, and break up text. But adding an image isnt just about slapping an <img> tag on the page. It involves choosing the right file format, optimizing for speed, and ensuring accessibility.

The Hero Image

A Hero Image is a large, prominently placed banner image at the top of a webpage. It is usually the first visual element a visitor encounters, making it crucial for setting the tone and capturing attention.

Understanding Image Compression
  • Lossy Compression (JPG/WebP): Mathematical data is permanently deleted to reduce the file size. This is perfect for large, complex photographs because the human eye can't notice the slight loss in quality.
  • Lossless Compression (PNG/GIF): The file size is reduced by rewriting the data without losing visual quality (similar to zipping a folder). Best for sharp graphics and text.
  • Dithering: An old technique of simulating colors using patterns of dots. It was used when color palettes were heavily restricted on early computers to prevent banding.
Quick-Reference Table: Image Formats
Format Full Name Best For Key Feature(s)
JPG Joint Photographic Experts Group Photos. Real-world pics. Small file size (Lossy compression)
PNG Portable Network Graphics Logos, Icons, Graphics Transparency support (Lossless)
GIF Graphics Interchange Format Memes. Short animations. Looped animation
SVG Scalable Vector Graphic Logos & Icons. The GOAT. Perfectly sharp at any size (Math-based Vector)
WebP WebP (Google) Modern Web Performance Superior lossless and lossy compression on the web

Key Takeaway: Use JPG or WebP for photos. Use SVG for all logos and icons. Use PNG only if you need a transparent photo.

Adding an Image to Your Website

To add a basic image, use the <img> tag. This is an "empty" or "self-closing" tag, meaning it does not wrap around text and does not require a closing </img> tag.

<img src="media/filename.jpg" alt="A descriptive text for screen readers">
The Rules of Alt Text: If an image fails to load, the browser will display the alt attribute's text instead.
  • When to use it: If an image contains important information, you MUST describe it (e.g. alt="A chart showing a 20% increase in sales").
  • When to leave it empty: If an image is purely decorative (like a background swirl), you must leave it empty (e.g. alt="") so screen readers skip it instead of confusing the user.

Pro Skill: Optimizing Images in Photoshop

You should NEVER use a 5MB photo straight from your phone or a modern digital camera on your website. If an image is 4000px wide, it is a terrible practice to simply use CSS width: 400px; to make it look smaller on screen. Why? Because CSS only changes how it looks; the browser still has to download the massive original file, slowing down your site to a crawl.

The 'Why': Real-world photos are massive. If you force a user's phone to download a heavy 5MB image just to show a small 400px thumbnail, they will likely leave your site before it loads. Compressing images ensures lightning-fast load times and preserves your bandwidth.

To fix this, professional web developers always optimize their images before writing a single line of HTML. Optimizing an image means finding the perfect balance between a tiny file size and high visual quality. Here is the standard three-step workflow we use in Adobe Photoshop:

Let's Discuss:

Think about a user on a slow mobile connection. How does properly cropping, resizing, and compressing a 5MB image down to 150KB impact their UX?

1. Crop (The “Get-to-the-Point” Tool)
  1. Open your image in Photoshop.
  2. Select the Crop Tool (C).
  3. Drag the handles to frame the most important part of the image. Press Enter.
2. Resize (The “Shrink Ray”)

We use the Image Size tool in Photoshop specifically because it permanently shrinks the total pixel dimensions of the image.

  1. Go to Image > Image Size... (Alt+Ctrl+I).
  2. Make sure the “chain link” icon is on (to keep the proportions).
  3. Change Dimensions to Pixels.
  4. Enter your target width (e.g., 800 pixels for a blog post).
  5. For Resample, choose Bicubic Sharper (reduction).
  6. Click OK.
3. Compress (The “Squoosher”)
  1. Go to File > Export > Save for Web (Legacy)... (Alt+Shift+Ctrl+S).
  2. This window is the secret sauce. Choose your format:
    • JPEG High: Perfect for photos. Lower the Quality slider (e.g., 60-80) and watch the file size drop.
    • PNG-24: Use this only if you need transparency.
  3. Check “Convert to sRGB” for good colors.
  4. Click Save... and name your new, optimized file (e.g., hero-image-optimized.jpg).
This process can turn a 5MB monster photo into a 150KB lightning-fast image.

Next Level: Responsive Images with srcset

Standard <img> tags are “one size fits all.” srcset turns your image into a smart element that adjusts based on the user's hardware. It lets you provide multiple versions of an image so the browser can pick the one that fits the screen size perfectly.

The 'Why': Standard <img> tags force all devices to download the exact same file. srcset is a responsive superpower that lets a mobile phone download a tiny version of an image while a 4K desktop monitor downloads the high-res version, saving immense bandwidth.
The Anatomy of a Responsive Image:
  • The srcset attribute: This is your menu of options. You list the file path and its actual width in pixels using the w descriptor (e.g., 800w).
  • The sizes attribute: This is the instruction manual for the browser. You tell the browser how much space the image will take up on the screen at different breakpoints.
  • The src fallback: You still include a standard src attribute so that older browsers that don't understand srcset still show an image.
Adding Responsive Images to Your Website
<img src="media/fallback.jpg" alt="Description"
    srcset="media/small.jpg 480w, media/large.jpg 800w"
    sizes="(max-width: 600px) 480px, 800px">

Vector Graphics (SVG & Illustrator)

What happens to a photo (JPG) when you zoom in 500%? It gets blurry and "pixelated" because it is a Raster image made of a grid of tiny colored squares.

Logos and icons require a different approach. We use SVG (Scalable Vector Graphics). SVGs are not made of pixels; they are made of mathematical paths, coordinates, and curves. Because it's just math, you can scale an SVG to the size of a billboard and it will remain perfectly crisp, all while having a file size of just a few kilobytes.

Creating an SVG in Adobe Illustrator

We use Adobe Illustrator to draw these mathematical paths using tools like the Pen Tool and the Pathfinder.

  1. Open Illustrator and create a new Web canvas.
  2. Use the Text tool or Shape tools to design a simple, bold logo.
  3. Select your text and choose Type > Create Outlines. This turns your font into raw mathematical paths so it loads correctly on any computer.
  4. Use the Pathfinder Tool to merge overlapping shapes into a single clean icon.
  5. Go to File > Export > Export As... and choose SVG.
Need a visual guide? Check out this Illustrator Beginner Tutorial for Simple Letter Logos.
The Text Editor Trick: Inlining SVG

If you have a .svg file on your computer, you can right-click it and choose Open with > Notepad (or drag it into VS Code). You will see the actual <svg> tags and path data, which you can then copy and paste directly into your HTML. By inlining the code, we can style it using CSS!

<div class="brand-logo">
    <svg xmlns="https://www.w3.org/2000/svg" ...>
        <!-- entire code from file -->
    </svg>
    <span>Galactic Gliders</span>
</div>

Favicons: The Tiny Details That Matter

A Favicon (short for "favorite icon") is the tiny logo that appears next to your website's title in the browser tab. While small, it is absolutely critical for branding, making your site look professional, and helping users find your tab when they have 50 of them open!

The 'Why': Brand logos must look perfectly crisp whether they are on a tiny business card or a massive billboard. SVGs use math instead of pixels to achieve infinite sharpness. Favicons are crucial for brand recognition when users have 20 tabs open.

In modern web design, we don't just use one favicon. Because users view websites on desktops, iPhones, and Androids, we have to provide a few different formats so the device can pick the best one. Here are the formats you need to know:

.ico Classic Fallback

Every browser looks for favicon.ico by default. It's a special format that holds multiple sizes (like 16x16 and 32x32) in one file. Used as a safety net for older browsers.

.png Modern Standard

Supports high-quality transparency. Most modern browsers prefer grabbing a 32x32 or 16x16 PNG over an .ico because they are sharper and easier to export.

.svg The Future

Drawn with math, an SVG favicon scales perfectly without blurring. You can even use CSS inside it to change colors for Dark Mode!

Apple Touch Icon

When an iOS user clicks "Add to Home Screen", Apple looks for a 180x180 PNG called apple-touch-icon.png to use as the app icon.

Android / PWA 192x192 .png

Android devices look for larger PNGs (192x192, 512x512) linked inside a .webmanifest file to generate home screen app icons.

Converting Your Logo to All These Formats

Here's the reality: you're not going to manually create 8 different versions of your logo in Photoshop. That's where Real Favicon Generator comes in. This free online tool does the hard work for you:

  1. Go to realfavicongenerator.net
  2. Upload your square logo image (PNG, JPG, or SVG)
  3. The tool automatically converts it into all eight formats you saw above. It generates all the HTML code you need (you'll paste it into your <head>)
  4. Create a new folder called favicon inside your media folder (or whatever you named the folder where all your images and media go).
  5. Download the package, extract the files, and place them ALL inside this new favicon folder.

Why do we need all these versions? Because different devices, operating systems, and browsers look for favicons in different ways. Your desktop browser wants a PNG. Your iPhone wants a specific size and file format. Android wants it referenced in a manifest file. By providing all formats, you guarantee your brand icon displays correctly everywhere.

Adding the Favicons to Your Website

You don't need to memorize these tags! You can place all of these right inside the <head> of your HTML document so the browser knows exactly where to find your icons within that new `favicon` folder:

<!-- Apple Touch Icon (iOS Home Screen) -->
<link rel="apple-touch-icon" sizes="180x180" href="media/favicon/apple-touch-icon.png">

<!-- Modern PNG Favicons -->
<link rel="icon" type="image/png" sizes="32x32" href="media/favicon/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="media/favicon/favicon-16x16.png">

<!-- Android Manifest -->
<link rel="manifest" href="media/favicon/site.webmanifest">

<!-- Fallback Classic ICO -->
<link rel="shortcut icon" href="media/favicon/favicon.ico">

Video (The Easy Way vs. The Pro Way)

The <iframe> tag allows you to embed external content like a YouTube video. This is the "Easy Way." Wrapping it in a div allows you to control responsiveness using the modern aspect-ratio property.

The 'Why': Video files are huge bandwidth hogs. By embedding a YouTube video or Google Map using an <iframe>, you are forcing Google's massive servers to handle the heavy processing and bandwidth costs instead of your own.

Why use a YouTube <iframe>? Because video files are huge! By embedding a video, you allow YouTube's massive servers to handle the heavy processing, saving you from paying for the massive bandwidth the video uses.

Adding an Iframe (YouTube) to Your Website

Pro Skill: The Native Media Environment

While embedding YouTube videos is best for saving bandwidth, sometimes you are required to host raw media files directly on your server (like building a custom Netflix clone or an internal company dashboard).

Because downloading and hosting massive media files is restricted on many school networks to save bandwidth, we are going to learn the native syntax by acting as QA Debuggers.

The 'Why': Even if you don't host large videos yourself due to server limits, as a developer you must know how to read, write, and debug standard HTML5 media tags—especially the strict accessibility rules around muted autoplay.
Essential <video> Attributes

Native HTML5 allows us to host media files directly. However, browsers have strict rules to protect users from annoying experiences. For example, if you want a video to autoplay, you must include the muted attribute. Browsers block noisy videos from playing automatically.

Attribute Description
controls Adds the play, volume, and CC buttons. Required for accessibility.
autoplay Playback starts immediately. Required: must be paired with muted.
muted Turns off audio by default. Often required for autoplay.
loop Restarts the video automatically so it loops forever.
poster Specifies a static thumbnail image to show before the user hits play.
Cultural Competency: The <track> Tag

If your video has spoken words and you provide controls, you have a legal and ethical obligation to provide subtitles for deaf and hard-of-hearing users. We achieve this by linking a .vtt (Web Video Text Tracks) file using the <track> tag.

Crucial Tech Note: Why We Need a Local Server (MAMP)

If you simply double-click your HTML file to open it in Chrome (using a file:/// path), your subtitle tracks will not load. Why? Browsers have strict security rules (CORS) that block local files from accessing other local files to prevent hackers from reading your hard drive.

What is MAMP? MAMP is a free program that turns your own computer into a simulated web server. Instead of reading a local file, you are accessing your site via http://localhost:8888.

Alternatives: If you don't have MAMP installed, you can easily bypass this issue by installing the "Live Server" extension directly inside VS Code, which launches its own temporary local server with one click!

Correct Native Video Syntax

The <source> tag is used inside your video or audio elements to provide fallback file formats.

<video controls poster="media/poster.png">
    <source src="media/drone-swarm.mp4" type="video/mp4">
    <track src="docs/subtitles.vtt" kind="subtitles" srclang="en" label="English" default>
</video>

Adding Audio (Podcasts & Sound FX)

Sound is the “hidden” layer of web design. It can be used for long-form content like podcasts or sound effects.

The 'Why': Browsers are picky about audio formats. You need to know how to properly construct fallback architectures (like .mp3 and .ogg) so your audio player functions flawlessly regardless of whether the user is on an iPhone or a Windows desktop.
The Anatomy of an <audio> Element
  • The Wrapper: The <audio> tag holds settings like controls.
  • The Source: The <source> tag tells the browser where the file is located.
  • The Fallback: Text between the tags only appears if the browser is ancient.
Audio Format Comparison
Format Strengths Weaknesses
MP3 Compatibility Lower quality than modern formats.
OGG Open-source Support can vary on older iOS devices.
WAV Highest quality Huge file size. Slows down loading.
Correct Native Audio Syntax
<audio controls preload="none">
    <source src="media/regulations.ogg" type="audio/ogg">
    <source src="media/regulations.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Old-School Cool: HTML Tables

Warning: Never use HTML tables for an entire page layout! In the early 2000s, developers used tables to build the grid of a website. This is a terrible practice because it is entirely inaccessible to screen readers and completely breaks responsive design. Today, tables are strictly for displaying tabular data (like schedules, pricing tiers, or statistics).

The 'Why': Never use tables for layout, but always use them for tabular data (like pricing, statistics, or schedules). Using proper semantic tags (<thead>, <tbody>, <th>) ensures screen readers can mathematically decipher the grid for visually impaired users.
The Anatomy of a Table

A semantic table is built like a sandwich:

Tag Definition
<table> Main container for all table data.
<caption> Acts as a title or explanation specifically for the table.
<thead> Header section.
<th> Table Header. Semantically correct for defining the header row containing column titles.
<tbody> Main data section containing regular rows.
<tfoot> Summary or totals section wrapping the bottom.
Table Superpowers

Use colspan to stretch a cell horizontally across multiple columns. Use rowspan to stretch a cell vertically across multiple rows. We will use colspan="3" in our lab to stretch a footer across the entire width.

Adding a Data Table to Your Website
<table>
    <caption>Employee Schedule</caption>
    <thead>
        <tr>
            <th>Header</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Data</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="3">Summary</td>
        </tr>
    </tfoot>
</table>

CSS Styling for Media (Object-Fit)

Have you ever seen an image on a website that looks completely squished or stretched out? This happens when developers force an image with a vertical aspect ratio into a horizontal CSS box.

To prevent this, we use the magical CSS property object-fit. It tells the browser how the image should behave inside its container.

The 'Why': When you force a vertical photo into a horizontal box, it stretches and looks terrible. `object-fit: cover` fixes this instantly. Additionally, hover effects provide crucial UX feedback, telling the user without words that an element is interactive.
  • object-fit: cover; – The image will scale and crop itself perfectly to fill the entire box without ever stretching. This is the gold standard for hero images and profile pictures.
  • object-fit: contain; – The image will scale down until the entire picture fits inside the box, usually leaving blank letterbox space on the sides.
CSS Architect Reminders

A few key rules that power professional CSS design:

  • The rem unit is highly recommended for typography because it is strictly relative to the Root HTML element's font size, scaling perfectly for visually impaired users.
  • Always start your stylesheet with box-sizing: border-box;. This is the "Magic Reset" that changes how width and height are calculated, preventing borders and padding from expanding a box and breaking your layout!

To ensure images are mobile-responsive, we apply a global CSS rule to all images:

img, video {
    max-width: 100%;
    height: auto;
}

The Architect's Primer: Web Accessibility

As a Digital Architect, you don't just build pretty pages; you are legally and ethically responsible for the code you deploy. When dealing with rich media like images and videos, accessibility (a11y) becomes paramount.

Web Accessibility (a11y) Core Principles

If you build a building without a wheelchair ramp, it is illegal. The web is no different.

  • The POUR Principles: Guidelines ensuring a site is Perceivable, Operable, Understandable, and Robust. For example, to be Operable, a site must be fully navigable using just a keyboard.
  • Color Contrast: A site with light gray text on a white background immediately fails accessibility checks. The recommended tool for verifying this is the WebAIM Contrast Checker.
  • The Curb Cut Effect: Designing for the most marginalized users (like adding closed captions to videos) often improves the experience for everyone.
  • Legal Standards: The Web Content Accessibility Guidelines (WCAG) dictate how to make sites accessible. The standard, legally targeted level of compliance is Level AA.

Project 3: Media Integration & Audits

You have built the structure. Now it's time to inject the media. As you finalize your custom website, you must ensure your file paths are perfectly accurate. Check your "Assets" folder. Are all your filenames in lowercase with no spaces? Linux servers are case-sensitive; Image.JPG and image.jpg are two different files to a web server!

The Lighthouse Audit

Before launching a site, professionals run a diagnostic audit. Open your website in Google Chrome. Right-click, select Inspect, and navigate to the Lighthouse tab in DevTools. Run a report for "Desktop."

Lighthouse will grade your site out of 100 for Performance, Accessibility, Best Practices, and SEO. If your images are too large, your Performance score will tank. Use Analytical Reasoning to read the report and apply the suggested technical fixes.
