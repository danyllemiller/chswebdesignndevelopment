The Final Boss: Going Live

You have built the assets. You have written the code. You have learned the theory. Now, it is time to face the final boss of web development: The Public Internet.

Up until now, your websites have lived exclusively on "Localhost" (your own computer's internal server). You can see them, but no one else in the world can. What does the term 'Deployment' mean in web development? Moving files from a local private environment to a public live environment. To finish the game and prove your skills, you must deploy your projects to a real web server in the cloud.

In this chapter, we will cover the legacy methods of deployment, modern automated pipelines, the complexities of domain names, and the strict Quality Assurance (QA) testing you must execute before you hit the launch button.

The "Old School" Way: FTP Uploads

For decades, FTP was the only way to launch a website. What does FTP stand for? File Transfer Protocol. It is the standard protocol for transferring files over the internet.

How FTP Works

To use FTP, you need a specialized program. What is the name of the popular, legacy software program used as a digital 'moving truck' for FTP? FileZilla. You open FileZilla, log into your server, and manually drag your HTML/CSS files from your computer to the server's public_html folder.

The Danger: Why is the 'Old School' FTP method considered dangerous for modern team development? It is manual, error-prone, and can instantly overwrite live files with no 'undo' button.

The "New School" Way: Git & CI/CD

Modern developers hate dragging and dropping files manually. The industry standard is CI/CD. What does CI/CD stand for? Continuous Integration / Continuous Deployment.

What is the primary benefit of a CI/CD pipeline? It automatically updates the live website seconds after you push code to GitHub. Why is 'Continuous Deployment' safer than manual FTP? It relies on version control, meaning every update is tracked and can be instantly rolled back if a bug occurs. When using hosts like Netlify, what branch does it monitor by default for automatic deployments? The 'main' or 'master' branch.

Webhooks & CDNs

How does a modern cloud host know when you have updated your code on GitHub? Through an automated digital tripwire called a Webhook. When the webhook fires, Netlify pulls your code and distributes it across a CDN. What is the primary function of a Content Delivery Network (CDN)? To host copies of your website on servers globally so it loads fast for everyone.

Automation Warning: A CI/CD pipeline does exactly what you tell it to do. If you push a broken index.html file to GitHub, what will the CI/CD pipeline do? Automatically deploy the broken code to the live site.

Debugging the Build: Common Launch Errors

Before moving on, we must address the most common reasons student portfolios fail to load on launch day.

The Path Paradox

What is the danger of hardcoding an Absolute Path (like C:/Users/Hannah/Desktop/image.jpg) in your HTML before deployment? It works on your laptop, but will break completely when moved to a cloud server. The server does not have a "Hannah" folder! Always use relative paths.

Index & Missing Files

Which file is automatically recognized as the default homepage by almost all web servers? index.html. If you name your homepage main.html, your live site will just show an ugly directory folder list.

Furthermore, what happens if you deploy an HTML file but forget to deploy the images folder that goes with it? The images will show as broken links (404 errors) on the live site. What is a '404 Error'? Not Found (The requested page or file does not exist on the server).

Pro Tip:

Case Sensitivity: Windows computers don't care about capitalization, but Linux cloud servers do! Logo.PNG and logo.png are two entirely different files on the web. Check your capitalization!

Owning Your Identity: Domains & DNS

Right now, your site lives on a free subdomain like hannah-dev.netlify.app. What is a 'Subdomain'? A prefix added to a domain name to separate content (like 'blog.mysite.com' vs 'mysite.com'). However, a custom domain projects professionalism.

How does a Custom Domain (like johndoe.com) improve a portfolio? It projects professionalism, authority, and brand ownership. When you purchase a domain name from a Registrar (like GoDaddy), how long do you typically 'own' it before having to renew? One Year.

The DNS Phonebook

What is the Domain Name System (DNS)? The internet's master phonebook that translates human-friendly names into computer-friendly IP addresses. To connect your custom domain to your host, you must edit your DNS records:

  • What is a DNS 'A Record' used for? Pointing a domain name to a specific numeric IP address.
  • What is a DNS 'CNAME Record' used for? Pointing a domain to another domain or subdomain URL (like a Netlify app link).
  • If you change a DNS setting, why might your friend in another city not see the change immediately? The change has not fully propagated across the global DNS network yet. What does 'DNS Propagation' refer to? The time it takes for a new domain setting to spread to every internet provider across the globe.
Security: HTTP vs HTTPS

What is the primary difference between HTTP and HTTPS? HTTPS encrypts the data being transferred; HTTP sends it in plain text. What is the purpose of an SSL/TLS Certificate on a live website? To enable the padlock icon and encrypt data sent between the user and the server.

Pro Tip:

Vendor Lock-in: What is 'Vendor Lock-in'? When you build a site on a closed platform (like Wix) and cannot easily move your code to a different host. By buying your own domain and using GitHub, you stay independent!

Quality Assurance (QA) & Auditing

In the context of launching a site, what does QA stand for? Quality Assurance. Never send an untested site to an employer.

  • 1. The "Resize" Test (Responsiveness)

    Why should you perform the 'Resize Test' before launching? To ensure the layout doesn't break and remains fully responsive on mobile-sized screens. If your QA test reveals that your website requires horizontal scrolling on a phone, what CSS feature do you need to fix? The Media Queries and responsive widths.

  • 2. The "Link Rot" Test

    What is 'Link Rot'? When hyperlinks point to pages or resources that have been deleted or moved (resulting in 404 errors). When an external link opens in the same tab and navigates the user away from your portfolio, it breaks best practices. What attribute fixes this? target="_blank".

  • 3. The Accessibility & Console Audits

    When conducting an 'Alt Audit' for accessibility, what are you checking? That every <img> tag has a descriptive alt="" attribute for screen readers. What should you be looking for when you open the Browser Developer Console during QA testing? Bright red JavaScript errors or failed image loads. If a user reports that a button on your live site doesn't work, what is the first debugging step you should take? Open the Browser Console to check for JavaScript errors.

  • 4. Google Lighthouse

    What is Google Lighthouse? A free, professional auditing tool built into Chrome that grades your site's performance and accessibility. What does the 'Performance' score primarily measure? How fast the page loads and becomes interactive. What does the 'SEO' score measure? Search Engine Optimization; how well search engines can understand and rank your page. If your QA test reveals that your text is light gray on a white background, which Lighthouse score will drop? Accessibility (due to low color contrast).

Your Trophy Case: The Professional Portfolio

Why is a portfolio so critical for a developer seeking a job? It acts as a 'Show, Don't Tell' trophy case that proves you can actually build functioning software.

The Architecture of a Portfolio
  • 1. The Case Studies: Why is a 'Case Study' better than just a screenshot? It explains your thought process, challenges faced, and how you solved them, which employers value highly. What is the 'STAR Method' used for in a portfolio Case Study? Structuring project descriptions by explaining the Situation, Task, Action, and Result. What does the 'A' stand for? Action (What you specifically did to solve the problem). What does the 'R' stand for? Result (The final outcome of your actions).
  • 2. The Code Links: Why is it important to include a link to the GitHub Repository for your portfolio projects? So senior engineers and hiring managers can read your raw code and see how you solved problems.
  • 3. Contact Info: Why do hiring managers look for a 'Contact Information' section on a portfolio? Because if they can't easily figure out how to email you, they will move on to the next candidate.
  • 4. The Resume: Why is it recommended to include a downloadable PDF resume on your portfolio? So recruiters can print it or upload it directly into their Applicant Tracking Systems (ATS).
The Mindset: Defeating Imposter Syndrome

It is entirely normal to feel like you don't know enough to apply for a job. What is 'Imposter Syndrome'? A psychological feeling that you aren't a 'real' professional or aren't good enough, despite your accomplishments. What is the best way to handle 'Imposter Syndrome'? Accept that all developers feel it, keep learning, and proudly share the work you have completed.

Why should you never wait for a project to be 'perfect' before putting it in your portfolio? Software is never truly finished; it is meant to be iterated upon. Launch it, learn from it, and improve it.

The Final Check
What is the very last thing you should do before officially sharing your portfolio link with an employer? Click every single link yourself to ensure absolutely nothing is broken.
