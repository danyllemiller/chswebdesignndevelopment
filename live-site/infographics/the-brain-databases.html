2. The Storage Showdown

SQL (Relational)

The "Organizer"

  • Rigid Tables & Columns
  • Strict Database Schemas
  • ACID Compliant (Reliable)
  • Best for Banking/Accounts
NoSQL (Non-Relational)

The "Scrapbook"

  • Flexible Documents
  • Stored in Collections
  • No predefined columns needed
  • Best for Real-time Chat/Social
Local Storage

The "Sticky Note"

  • Client-Side Browser only
  • 5 MB Capacity Limit
  • Saves only Strings (Text)
  • Vanish if cache is cleared

3. SQL Language & CRUD

SELECT username, score_value
FROM HighScores
WHERE game_title LIKE 'Cosmic%'
ORDER BY score_value DESC
LIMIT 10;

Syntax Key: % is a wildcard for partial matches. DESC sorts High to Low. LIMIT restricts the result count.

Security Protocol

  • SQL Injection (SQLi): Attackers type code into inputs.
    The Fix: Parameterized Queries (sanitization).
  • Hashing: One-way math meat-grinder for passwords. Professionals use bcrypt or Argon2.
  • Salting: Adding random noise before hashing to stop Rainbow Table attacks.

The Pro Toolkit

  • ACID Compliance: Guarantees transactions succeed or fail as a whole (Atomicity, Consistency, Isolation, Durability).
  • Index: A special structure that massively speeds up search queries on specific columns.
  • ORM: Object-Relational Mapper; lets you write JS instead of raw SQL.
  • Data Validation: Checking data (like @ in email) before saving. Must happen on Client and Server.
