Tech Talk: Chapter 13 (The Cloud)

AWS (Amazon Web Services)
The largest and original cloud provider, hosting platforms like Netflix and Disney+.
Azure (Microsoft Azure)
The corporate choice for cloud, deeply integrated with Windows and Microsoft Office products.
Branch
An isolated copy of a project where a developer can safely work without affecting the main code.
Commit
A permanent snapshot of code saved to the local history.
Example: git commit -m "Added nav bar"
Data Center
A physical warehouse filled with thousands of servers, cooling systems, and backup generators.
Deploy
Moving code from a local development folder to a live, public server for the world to see.
Elasticity
The ability for a cloud server to automatically shrink back down after a traffic spike to save money.
GCP (Google Cloud Platform)
Google's cloud infrastructure, famous for high-speed AI and machine learning tasks (hosts YouTube).
git clone
Downloads a full copy of a GitHub project to your local machine for the first time.
Example: git clone https://github.com/user/project
git pull
Downloads the latest changes from a teammate's remote work to your local hard drive.
Example: git pull origin main
IaaS (Infrastructure as a Service)
Renting raw hardware (Virtual Machines). You install the OS and manage the software.
Merge Conflict
When two developers edit the same line of code simultaneously and the computer doesn't know which one to keep.
PaaS (Platform as a Service)
Renting a "Kitchen for Code." You bring the script; the host (like Netlify) handles the server management.
Public Cloud
Hardware owned by a provider (AWS) and shared among thousands of different customers.
Repository (Repo)
The main project folder containing all code files and the hidden Git version history.
SaaS (Software as a Service)
Using finished software online (Google Docs, Netflix) where the provider manages everything.
Scalability
The ability to easily handle a growing amount of traffic by adding more computer power.
VCS (Version Control System)
Software (like Git) that acts as a "Time Machine" for code, tracking every change made by a team.
