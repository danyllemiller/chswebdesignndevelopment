Tech Talk: Chapter 16

Exhaustive Vocabulary for Databases & SQL Persistence

ACID Compliance
Atomicity, Consistency, Isolation, Durability. A set of properties that guarantee database transactions are processed reliably, essential for financial and critical data systems.
bcrypt / Argon2
Modern, secure mathematical algorithms used for Hashing passwords. They are specifically designed to be slow to compute, protecting against brute-force attacks.
Boolean Field
A database column type that only stores two possible values: True (1) or False (0).
CRUD
Create, Read, Update, Delete; the four fundamental functions of any persistent storage system.
Data Normalization
The process of organizing data in a database to reduce redundancy (repetition) and improve integrity using primary and foreign keys.
Data Validation
Checking if data meets specific rules (e.g., age > 13) before it is saved. For security, this must happen on both the Client-Side and Server-Side.
DROP TABLE
The catastrophic SQL command that permanently deletes an entire table's structure AND all its data from the database.
Foreign Key
A column in one table that points back to the Primary Key of another table, creating a relational link between them.
Hashing
A one-way mathematical function that turns a password into a unique "fingerprint." It can never be reversed, only compared for a match.
Index
A specialized data structure that massively speeds up search queries on a column, much like an index in a physical book.
JOIN
The SQL command used to combine rows from two or more tables based on a related column between them.
LIKE % (Wildcard)
An SQL operator used for partial string matching. The % symbol represents any sequence of characters (e.g., 'Co%' finds 'Cody' and 'Comet').
Local Storage
A 5MB segment of memory located entirely in the user's browser. It is non-persistent across devices and only stores Strings.
NoSQL
Non-relational databases (like Firebase) that store data in flexible "Documents" and "Collections" rather than rigid tables.
ORM (Object-Relational Mapper)
A tool (like Sequelize) that allows developers to interact with a database using their programming language (JavaScript) instead of raw SQL code.
Parameterized Query
A secure coding method that "sanitizes" user input by treating it as harmless text rather than executable code, stopping SQL Injection.
Primary Key
A unique identifier for every single row in a table. It must be unique and cannot be null.
Rainbow Tables
Pre-computed lists of common password hashes. Developers use Salting to render these tables useless.
Salt
A string of random characters added to a password before it is hashed, ensuring two users with the same password have unique hash fingerprints.
Schema
The structural blueprint or "map" that defines the tables, columns, and data types of a relational database.
SQL Injection (SQLi)
A dangerous attack where a hacker types SQL code into an input field (like a search box) to trick the database into performing malicious tasks.
