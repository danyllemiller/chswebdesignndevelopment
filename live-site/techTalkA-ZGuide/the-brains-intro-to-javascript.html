Tech Talk: Chapter 11

Exhaustive Vocabulary for JavaScript Logic & DOM Manipulation

addEventListener()
The standard method used to attach an event handler (a "watchman") to an HTML element, listening for actions like click or submit.
Array ( [ ] )
A digital "backpack" or list that stores multiple items in a specific order using zero-based indexing.
Assignment Operator (=)
The symbol used to store a value inside a variable "box." (Note: This is different from the comparison ===).
Boolean
A primitive data type that can only have one of two values: true or false.
Client-Side Scripting
Code (JavaScript) that runs directly inside the user's web browser, providing instant interactivity without a server round-trip.
Comparison Operators
Symbols used to evaluate logic, including === (strict equality), !== (inequality), >, and <.
Conditional Logic (if/else)
The core of decision-making in code; allows different blocks of instructions to run based on whether a condition is true or false.
const (Constant)
A keyword used to declare a variable that cannot be re-assigned. Used for HTML elements and fixed values to prevent TypeError.
DOM (Document Object Model)
The browser's internal "family tree" or map of an HTML page, which JavaScript uses to find and manipulate elements.
DOMContentLoaded
The event that fires when the HTML is completely loaded and parsed, ensuring the "brain" doesn't try to work before the "bones" exist.
Dynamic Website
A site that reacts to user input and changes content "on the fly" using scripting.
Event Bubbling
The process where an event triggered on a child element (like a button) moves up through all of its parent elements in the DOM.
Function
A reusable block of code designed to perform a specific task; often referred to as a "Smoothie Machine."
let
A keyword used to declare a variable that can change its value later (e.g., a score counter or timer).
Logical Operators (&&, ||, !)
Symbols used to combine logic: AND (&&), OR (||), and NOT (!).
Loop (for...of)
A control structure used to repeat instructions; for...of is specifically used to iterate through the items in an Array.
Modulus (%)
An arithmetic operator that returns the remainder of a division problem (e.g., 10 % 3 = 1).
Null vs. Undefined
Null: An intentional empty value.
Undefined: A variable that was declared but hasn't been given a value yet.
Object ( { } )
A digital "trading card" that stores related data in descriptive key: value pairs.
Parameter
The "ingredients" or variables listed in a function's definition that it needs to perform its job.
preventDefault()
A method used inside event listeners (usually submit) to stop the browser's default action, like refreshing the page.
querySelector()
The primary DOM method used to search for and select the first HTML element that matches a specific CSS selector.
Return
The keyword that stops a function and sends a specific value (the "finished smoothie") back to where the function was called.
Scope (Global vs. Local)
Rules defining where a variable can be seen. Local variables inside functions are invisible to the rest of the script.
Strict Equality (===)
The professional standard for comparison that checks both the Value and the Data Type.
Template Literal (Backticks)
Strings wrapped in ` ` that allow for easy variable injection using the ${} syntax.
Test Alert!

Q: What is a ReferenceError?

A: An error that occurs when you try to use a variable that hasn't been declared or doesn't exist.

Q: What is the result of "10" + 5?

A: "105" (Concatenation) because one value is a string!
