Computers are incredibly fast, but on their own, they are incredibly dumb. If you give a computer a list of instructions, it will run them from top to bottom, one by one, and then stop. But what if we only want an instruction to run sometimes? What if we want to run an instruction 1,000 times? In this chapter, we explore Control Structures—the tools we use to manipulate the flow of an algorithm to make our programs dynamic and intelligent.

12.1 Conditionals & Boolean Logic

The most fundamental control structure is the decision. By using "Conditionals," we can write programs that evaluate the current state of the world and choose different paths based on what they find.

12.1.1 The "If-Then" Decision

An If-Then statement allows a computer to evaluate a condition. A condition is simply a question that has a True or False answer. If the answer is True, the computer runs a specific block of code. If the answer is False, the computer ignores that code completely.

Real-World Logic

Your smartphone is constantly running conditional statements in the background:

IF (batteryLevel < 20) {
    turnOnPowerSaveMode();
    dimScreenBrightness();
}

12.1.2 Boolean Operators (AND, OR, NOT)

What if a decision requires checking more than one thing at the same time? We use Boolean Operators to combine multiple conditions into a single, complex rule.

AND (&&)

BOTH conditions must be True.
"If I have $10 AND the store is open, I will buy a snack."

OR (||)

ONLY ONE condition must be True.
"If it is Saturday OR it is Sunday, I will sleep in."

NOT (!)

REVERSES the condition.
"If it is NOT raining, I will walk to school."

12.1.3 Nested Logic

Sometimes, passing one condition just leads to another question. Placing one decision inside of another is called Nested Logic. This allows for advanced, multi-layered problem solving.

Step-by-Step: Writing a Nested IF Statement

Let's write pseudocode for a video game checking if a player can open a treasure chest.

  1. The First Check: IF player_is_near_chest == True:
  2. The Nested Check: We indent our code to show it's inside the first block.   IF player_has_key == True:
  3. The Result:     open_chest()
  4. The Fallback (Else):   ELSE:
        print("You need a key!")

The computer will only ask if the player has a key IF they are already standing next to the chest. This saves processing power!

12.2 Loops & Iteration Standard 9-12.AP.C.1

If you want a video game character to take 10 steps forward, writing the command step_forward() ten separate times is incredibly inefficient. Instead, we use Loops to iterate (repeat) code automatically.

12.2.1 Why Repeat? (For vs. While)

There are two primary types of loops, and choosing the right one depends on whether you know how many times the code needs to run.

FOR Loops

A For Loop repeats a block of code a set number of times. You use this when you know the exact end goal.

Example: "For exactly 10 times, move the character forward 1 step."

WHILE Loops

A While Loop repeats code continuously until a specific condition changes. You use this when you don't know how long it will take.

Example: "While the player's health is greater than 0, keep playing the background music."

12.2.2 Tradeoffs in Implementation

Why do we use loops? In computer science, there is a principle called DRY: Don't Repeat Yourself. Writing the same line of code over and over (copy-pasting) makes your file size massive, makes it hard to find errors, and is a nightmare to update. If you need to change the character's step size from 1 to 2, in a copied list you have to change it 10 times. With a loop, you only have to change it once.

12.2.3 Readability vs. Performance Standard 9-12.AP.C.1

When you write code, you are actually writing for two audiences: the computer (which needs it to run fast) and other humans (who need to be able to read and understand it later). You must be able to justify why you chose a specific control structure.

If you write an endlessly nested set of 20 If-Then statements, the computer can read it instantly. However, a human developer will look at that code and be completely confused. By organizing your logic into clean loops and breaking it into modular procedures (as we learned in Chapter 11), you sacrifice a tiny fraction of a millisecond of computer speed to gain massive human readability. In professional software development, readable, maintainable code is almost always preferred over confusing, slightly faster code.

12.3 Event-Driven Programming Standard 9-12.AP.C.2

So far, we have looked at code that runs from top to bottom. But modern apps (like TikTok or a web browser) don't just run and stop. They sit on your screen and wait for you to do something. This requires a completely different architectural mindset.

12.3.1 What is an Event?

Event-Driven Programming is a paradigm where the flow of the program is determined by user actions (Events). Instead of the computer dictating the pace, the user is in control.

An Event is any action that occurs in the system. Common events include:

  • Mouse Events: onClick, onHover, onScroll.
  • Keyboard Events: onKeyPress, onKeyUp.
  • System Events: onTimer (like a stopwatch hitting zero), or onLoad (when a webpage finishes downloading).

12.3.2 Event Handlers (The Listeners)

To make an event actually do something, we use an Event Handler (sometimes called an Event Listener). This is a specific block of code that sits silently in the background, listening for its assigned event to happen. Once it "hears" the event, it jumps into action.

The Anatomy of a Handler

If you have a "Submit" button on a webpage, you must attach an Event Handler to it:

submitButton.addEventListener("click", function() {
    // This code only runs when the mouse clicks the button
    checkPassword();
    logUserIn();
});

12.3.3 Practical Intent & Personal Expression Standard 9-12.AP.C.2

Event-driven programming is the bridge between a boring list of math equations and an interactive masterpiece. By combining Events with the Conditionals and Loops we learned earlier, we can design artifacts that actually respond to humans.

Computer science is not just about crunching data; it is a medium for Personal Expression. You can use event handlers to build a practical tool (like a calculator app that responds to button clicks) or a piece of interactive digital art (where moving your mouse across the screen changes the colors of a canvas using the onHover event).
