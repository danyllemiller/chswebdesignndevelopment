🎯 Essential Question:

How does a UI/UX designer use psychology and digital prototyping tools to create intuitive and aesthetically pleasing interfaces?

PART 1: The Psychology of Design
  1. Visual Hierarchy: Visual Hierarchy is the deliberate arrangement of elements to imply . You want to guide the user's eye to the Primary CTA (Call to ) first.

    We achieve hierarchy by manipulating Scale (Size) and (Color differences).

    Designers use the "Squint Test" to quickly evaluate if their hierarchy is working.
  2. Cognitive Load & Spacing:
    • • Cognitive Load: The amount of effort required by a user to understand an interface.
    • • White Space: Also known as negative space. This is the empty "breathing room" around elements used to reduce cognitive load.
PART 2: Color Theory & Typography
  1. Color Psychology: Colors evoke subconscious emotional responses.
    • • Blue: Often represents and security (used heavily by banks).
    • • Red: Often represents urgency, errors, or danger.
    • • The 60-30-10 Rule: A standard UI palette consists of 60% Primary Color, 30% Secondary Color, and 10% Color (used for buttons).
  2. Typography:
    • • : Fonts with small decorative "feet" or strokes at the ends of the letters (traditional/formal).
    • • Sans-: Fonts without the decorative feet. Generally considered more modern and cleaner for digital screens.
    • • Leading: The vertical spacing between lines of text (Line-Height).
    • • Kerning: The horizontal spacing between individual characters.
PART 3: Figma Foundations & Components
  1. UI Prototyping: While a Wireframe is a black-and-white sketch, a -Fidelity (Hi-Fi) Mockup includes the final colors, typography, and actual images. We use cloud tools like to build these without writing code.
  2. Professional Organization:
    • • The Layer Stack: The order of elements from front to back. A button's text must sit the background shape layer to be visible.
    • • Professional Decorum: You must name your design layers clearly (e.g., submit-btn). Leaving layers named "Rectangle 4" causes chaos for the developers writing the code.
  3. Reusable Assets: A reusable master element in Figma (like a standard navigation bar) is called a . If you update the 'Master' version, it automatically updates all instances of it across the entire design file.
  4. Interactive Prototyping: Prototyping allows you to create a clickable simulation of the final app. You connect screens using a (the user action, like a Click or Hover) that starts an animation or screen change.
