Tech Talk: Chapter 14

Exhaustive Vocabulary for Content Management & Digital Strategy

Administrator
The highest level of user role in a CMS with absolute total control over every aspect of the site, including themes, plugins, and user permissions.
Block Editor (Gutenberg)
The modern default WordPress editor where every piece of content (paragraph, image, video) is treated as a modular, movable "Block."
Caching
The process of creating static copies of dynamic web pages to serve them much faster to visitors, reducing server load and wait times.
CMS (Content Management System)
Software that facilitates creating, editing, organizing, and publishing digital content by separating it from the underlying code structure.
Custom HTML Block
A specific block type that allows Managers to write or paste raw HTML/CSS code directly into a page when the visual editor lacks a specific feature.
Dashboard
The private back-end mission control center accessed via /wp-admin where developers and managers configure the website.
Database (MySQL)
The "Brain" of the CMS. A structured digital filing cabinet that stores all the text, user accounts, and settings for the website.
Draft vs. Published
Drafts: Content saved in the back-end but hidden from the public.
Published: Content that is live and accessible to the world.
Featured Image
The primary thumbnail image assigned to a post, used by themes in archives and displayed automatically when the link is shared on social media.
Open-Source
A philosophy of software development where the source code is free and available for anyone to inspect, modify, and enhance.
Permalink
The permanent URL of a specific page or post. Professional managers configure these to be human-readable (e.g., /about-us) for better SEO.
Page
Timeless, permanent information that is not organized by date. Usually used for "About Us," "Contact," or "Privacy Policy."
PHP
The server-side programming language that powers the logic, templates, and core functionality of the WordPress engine.
Plugin
A piece of software containing a group of functions that can be added to a WordPress website to extend functionality (e.g., adding a Store or a Form).
Post
Timely content arranged in reverse-chronological order. Highly dependent on Categories and Tags for organization and searchability.
Revision
An automatically saved previous version of a post or page that allows a manager to "go back in time" and restore content if a mistake is made.
SEO (Search Engine Optimization)
The process of improving a site's visibility in search engines. CMS platforms help by generating clean code, sitemaps, and optimized metadata.
Shortcode
A small piece of code inside brackets (e.g., [gallery]) used to execute a larger script or plugin feature within a post.
Theme
A collection of templates and stylesheets used to define the visual design and layout of a website. Changing a theme changes the "Clothes" but not the data.
User Roles
A system of permissions (Admin, Editor, Author, Contributor) that restricts what a user can see or do in the Dashboard to protect site security.
Vendor Lock-in
A major drawback of "No-Code" builders (Wix/Squarespace) where you do not own the code and cannot easily move your site to a new host.
Widget
A small block that performs a specific function (like a search bar or calendar), often placed in global areas like sidebars or footers.
WooCommerce
The world's most popular e-commerce plugin that transforms a standard WordPress site into a fully functional online store.
The .com vs .org Rule
.com: The Rental model. Easy but limited.
.org: The Owner model. Professional standard with total freedom.
The Golden Rule
Before updating the core, themes, or plugins, a professional developer MUST perform a full Backup of files and the database.
