Document Flow & Display Types

By default, elements on a webpage follow Normal Document Flow. The browser reads your HTML from top to bottom and places elements on the screen based on their display type. Think of it like organizing items on a desk.

The display Property
Display TypeBehaviorExamples
block The "Hogs." A block element naturally takes up 100% of the available width and forces a new line after it. They stack on top of each other like bricks. <div>, <p>, <h1>, <section>
inline The "Polite Ones." They only take up as much width as their content needs and sit side-by-side like beads on a string. Critical Rule: You cannot set a defined width/height or top/bottom margins on inline elements! <span>, <a>, <strong>
inline-block The "Hybrid." It sits side-by-side in a line with other elements, but it respects width and height properties like a block. Buttons, Navigation Links
Visibility vs. Display

When you need to hide an element, you have two choices, and they affect the layout completely differently:

PropertyBehaviorFlow Impact
display: none;Completely removes the element from the document flow as if it doesn't exist.Adjacent boxes fill the space.
visibility: hidden;Invisible, but it reserves physical space.Leaves an empty hole in the layout.

The Box Model: Spacing Math

Every HTML element is a rectangular box. The Box Model describes the 4 layers of this box, from the inside out:

  • Content: The actual stuff (text, images).
  • Padding: The empty space inside the box. It acts like cushioning between the content and the border.
  • Border: The "wall" of the box.
  • Margin: The empty space outside the box. This is the element's personal space that pushes other elements away.
The Pro-Tip: The "Magic Reset" (box-sizing: border-box)

By default, browsers use box-sizing: content-box. In this mode, padding and borders are added to the width you specify. Example: If an element has 100px width, 10px padding, and a 5px border (using content-box), what is the total rendered width? You must add the padding and border of BOTH sides (left and right) to the original width: 100 + (10+10) + (5+5) = 130px total width. This breaks grid layouts easily!

To fix this, professionals use box-sizing: border-box;. This ensures that padding and borders are included in the element's total width and height, meaning a 100px box stays exactly 100px.

Layout Quirks & Tricks
  • Margin Collapse: When two vertical margins touch (like the bottom margin of an H1 and the top margin of a P), they don't add together. They collapse into a single margin equal to the larger of the two values.
  • Centering a Block: What is the classic CSS technique to horizontally center a block-level element (like a container div) with a set width? You use margin: 0 auto;. Setting left and right margins to 'auto' forces the browser to split the remaining space equally.
  • Overflow: If text is spilling out of its container, setting overflow: scroll; forces a scrollbar to appear inside the box.

Flexbox: 1-Dimensional Layouts

For years, developers used complicated "float hacks" to put boxes next to each other. Now, we use Flexbox (Flexible Box Module). Flexbox is designed for laying out items in a single row or a single column (1-Dimensional). It is the absolute gold standard for navigation bars, aligning icons, and perfectly centering content.

The Golden Rule of Flex

To use Flexbox, you must apply display: flex; to the Parent Container, not the child elements. Once the parent is a flex container, the immediate children become "flex items" and follow flexbox rules.

Core Flexbox Properties
PropertyWhat it Does
flex-direction Determines the Main Axis. The default of a newly created flex container is row (horizontal). You can change it to column to stack items vertically.
justify-content Aligns items along the Main Axis.
  • center: Packs items in the middle.
  • space-between: Pushes the first item to the start edge and the last item to the end edge, spacing the rest evenly.
  • space-evenly: Ensures the gap between all items AND the edges of the container are exactly identical.
align-items Aligns individual items along the Cross Axis (usually vertically). If align-items: center isn't vertically centering your flex item, the most likely cause is that the flex container doesn't have a defined height!
flex-wrap If a flex container runs out of horizontal room, items will normally shrink. To force them to drop to a new line instead, use flex-wrap: wrap;.
align-content Only matters if you have multiple wrapped rows! align-items handles individual items on a single line; align-content handles the spacing of multiple wrapped lines as a group.
flex-grow Applied to the child item. Determines how much of the available free space the item should take up.
order Applied to the child item. Allows you to visually re-arrange the sequence of flex items without changing the HTML markup (e.g., order: -1; moves it to the front).

CSS Grid: 2-Dimensional Architecture

While Flexbox is great for a single line of items, CSS Grid is primarily designed for 2-Dimensional layouts (Rows AND Columns simultaneously). It allows you to build the "skeleton" of your entire client webpage with incredible precision.

Defining the Grid and the "fr" Unit

To use it, apply display: grid; to a parent container. Then, define your columns and rows. Grid introduces a powerful new unit: fr (Fractional Unit). It represents a fraction of the available free space in the grid container.

  • 3 Equal Columns: grid-template-columns: 1fr 1fr 1fr; (or use the shorthand: repeat(3, 1fr)).
  • Row Heights: Which CSS property defines the height of rows? grid-template-rows: 200px 1fr;
  • Spacing: What does the gap property do? It sets the space between grid tracks (rows/cols) without using margins, keeping the math clean!
Advanced Grid Powers
Property / FunctionWhat it Does
grid-column: span 2; Tells a single grid item to stretch across two columns.
grid-column: 1 / 3; Does the exact same thing! It signifies the item starts at vertical grid line 1 and ends at grid line 3 (spanning 2 columns).
grid-template-areas Allows you to map out your layout using named visual strings (e.g., "header header" "sidebar content").
minmax(200px, 1fr) Defines a size range for a track, setting a minimum size (so it never squishes too small) and allowing it to grow to a maximum size.
auto-fill vs auto-fit Both create responsive grids without media queries.
auto-fill: Leaves empty tracks if there's extra space.
auto-fit: Collapses empty tracks to stretch existing items to fill the row.
Labs 4 & 5: Grid Skeleton & Gallery [50 pts]

Objective: Architect multi-dimensional layouts for your client's content organization.

Positioning & Z-Index

Sometimes you need to break an element out of the normal flow and place it exactly where you want it on the screen.

The position Property Values
ValueHow it affects the element
static The default. It ignores top, bottom, left, and right coordinate offsets completely.
relative Allows you to move it using top/left relative to its normal position, BUT it acts as an "anchor" parent for any absolute children inside it.
absolute Removes the element from normal document flow entirely. It is positioned relative to its closest non-static positioned ancestor.
fixed Glues an element to the viewport so it never moves when the user scrolls.
sticky Acts as 'relative' until it reaches a specific scroll threshold, then stays in place (like a sticky navbar).
Stacking Order: z-index

Which CSS property controls the stacking order of overlapping elements on the Z-axis (front-to-back)? z-index! Higher numbers (like 1000) sit on top. And yes, a z-index CAN have a negative value (e.g., -1), which pushes the element behind the default background layer.

Lab 6: The Sticky Header [25 pts]

  2. Apply position: sticky; and top: 0; to anchor it to the top.
  3. Give it a z-index: 1000; to ensure it stays on top of all other content (like images) as the user scrolls.
  4. Ensure the sticky element has a solid background color so the text underneath doesn't make the links unreadable!

Responsive Sizing: Stop Using Pixels!

What happens if a block element's width is set to 1000px, but the user's screen is only 400px wide? The browser creates an annoying horizontal scrollbar. To fix this, modern web design relies on relative units and mathematical functions.

The Relative Units
  • % (Percentage): If a child div is set to width: 50%, it will take up half the width of its direct parent element.
  • em: The em unit is relative to the font-size of its direct parent element.
  • rem: Relative to the root (<html>) element font size.
  • vw and vh: Viewport units. vw represents 1% of the browser window's width, and vh stands for Viewport Height.
Responsive CSS Functions
FunctionDefinition & Exam Hint
calc() Allows you to perform math directly in your stylesheet (e.g., subtracting pixels from percentages like width: calc(100% - 40px);).
clamp() Perfect for fluid typography. It allows font size to scale smoothly between a minimum floor and a maximum ceiling (e.g., font-size: clamp(1rem, 2vw, 3rem);).
aspect-ratio Maintains an element's proportions (like a 16:9 video) as it resizes responsively without distorting.
The Magic Image Fix

What is the standard "magic fix" to ensure images never overflow their parent containers? Always set max-width: 100%; height: auto; globally on all images.

Media Queries & Mobile-First Design

To truly change the layout based on the screen size, we use a Media Query. It is an "If-Then" statement for your website. If the screen matches a condition, then apply these specific CSS rules.

The Mandatory HTML Tag

Before any media query will work, you MUST have the viewport tag in your HTML head: <meta name="viewport" content="width=device-width, initial-scale=1.0">. What does it do? It tells mobile devices not to zoom out, and to render the page at its actual device width!

Mobile-First Strategy

The "Mobile-First" approach means coding styles for small screens first, then using min-width media queries to add complex layouts for larger screens. This forces you to prioritize content for users on the go.

/* Mobile-First: Default styles apply to phones */
.gallery { display: grid; grid-template-columns: 1fr; }

/* Desktop: If you use (min-width: 1024px), the CSS inside will apply to screens 1024px and wider. */
@media (min-width: 1024px) {
  .gallery { grid-template-columns: 1fr 1fr 1fr; }
}

/* Device specific triggers */
@media (orientation: landscape) {
  /* Detects if the user is holding their phone sideways */
  .hero { height: 100vh; }
}
Breakpoints

A "Breakpoint" refers to the pixel width where a media query triggers a layout change (e.g., 768px for tablets). The syntax to target screens 600px wide or smaller is: @media (max-width: 600px).

Labs 7-9: Breakpoints & Refactoring [75 pts]

Objective: Implement responsive breakpoints and refactor code to meet professional agency maintainability standards.

  1. The Viewport: Verify that the Viewport Meta tag is present in the <head> of all your HTML files.
  2. Breakpoints: Add a media query (e.g., @media (max-width: 768px)). Inside, change your Flexbox Nav from a row to a column. Change your multi-column Grid layouts back to a 1-column layout (grid-template-columns: 1fr;).
  3. Refactor: Go through your entire style.css. Ensure all hard-coded colors are using var(--variable) and replace fixed pixel font sizes with rem or clamp().
