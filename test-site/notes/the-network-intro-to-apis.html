Mastering the Request (CRUD)

The full power of an API is managed through four core HTTP Methods that map to the CRUD operations.

HTTP Method CRUD Operation Purpose (The Action) Analogy (The Waiter)
GET data from the server (most common front-end request). "I'd like the steak, please."
POST Send data to the server to create a resource (e.g., new user). "I want to submit my order."
PUT/PATCH an existing resource (e.g., change username). "I need to change my side dish."

III. Robustness & Error Handling
Status Codes (The Server's Reply)

The server always sends a numerical back with the response.

Code Range Meaning Client Action
2xx (e.g., 200 OK) Proceed to parse the JSON.
4xx (e.g., 404 Not Found) Error Stop and handle the error (don't try to read the data).
5xx (e.g., 500 Internal Server Error) Error Stop and handle the error.
The Safety Net
  • • try/catch: The block used for professional error handling. The risky await calls are wrapped in the block.
  • • Failure Scenario: If any line inside the try block fails (e.g., network failure, a 4xx error), execution immediately jumps to the block.
