THE ARCHITECT'S ROADMAP

Visualizing Structure, Logic & User Flow

1. The Structural Trio

SITEMAP
The Hierarchy

A tree diagram showing how pages connect. Prevents Orphan Pages and messy architecture.

WIREFRAME
The Blueprint

Low-fidelity lines and Lorem Ipsum. Focuses on structure, not colors or images.

USER FLOW
The Logic

The chronological sequence of steps a user takes to achieve a job. Shows decision points.

The Visual Scan (Z)

Users move Top-Left → Top-Right → Down-Left → Bottom-Right. Best for landing pages and hero images.

Z

The Content Scan (F)

Users scan across the top, then down the left margin, then across the middle. Best for text-heavy blogs.

F

3. Flowchart Logic Key

Rect.

Process: A specific action or task performed by the user or system.

Diam.

Decision: A fork in the road (Yes/No) that changes the user's path.

The 3-Click Rule

If it takes more than 3 clicks to find the goal, Cognitive Load is too high and users will leave.

Mobile-First

Prioritize core content for small screens first, then expand.

Content-First

The information (data) determines the size of the boxes.

Card Sorting

UX method where users group topics to build the Sitemap.
