Chapter 5: The "Bones" (Intro to HTML)

The TL;DR: What You Really Need to Know

The Core Concepts of HTML

  1. Code in a Text Editor: Always write your code in a specialized text editor (like VS Code), not a word processor (like Word or Google Docs).
  2. The Boilerplate is Essential: Every HTML file must start with the "boilerplate": <!DOCTYPE html> , <html> , <head> , and <body> .
  3. <head> vs. <body> : The <head> is for invisible metadata (like <title> , <meta charset> , <meta name="viewport"> , and <link> ing CSS). The <body> is for all the visible content.
  4. Tags and Attributes: Tags are "commands" ( <p> ). Attributes give more info to the opening tag ( <a href="..."> ).
  5. Semantic Layout: Use semantic tags for your page layout: <header> , <nav> , <main> , <article> , <aside> , and <footer> .
  6. Common Content Tags: The essentials are <h1> to <h6> for headings, <p> for paragraphs, <a> for links, and <img> for images.
  7. Images Need alt Text: Every <img> tag must have a src attribute (where to find the image) and an alt attribute (to describe the image for accessibility).
  8. Organize with Lists: Use <ol> for ordered (numbered) lists, <ul> for unordered (bulleted) lists, and <dl> for description lists.
  9. Links are Key: The <a> tag creates links. Its most important attribute is href , which specifies the destination URL (e.g., page.html , https://google.com , or mailto:email@example.com ).
  10. File Paths Matter: Use relative paths ( /folder/page.html ) to link between your site's pages so your links don't break.
  11. Special Characters: Use character entities (like &copy; for ©, &bull; for •, and &nbsp; for a space) to display special symbols.
