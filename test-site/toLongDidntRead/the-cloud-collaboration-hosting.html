TL;DR: Chapter 13

VCS Workflows & Cloud Infrastructure Essentials

The 4 Pillars of Modern Infrastructure

1. Version Control (The Engine)

Git is local software that acts as a code "time machine." GitHub is the cloud hub where you share that history. Professional teams use Branches to experiment without breaking the definitive Main/Master timeline. Always write descriptive Commit Messages to document changes.

2. The Service Models (Pizza Analogy)
  • IaaS: Raw hardware (Virtual Machines). You build the "house."
  • PaaS: A kitchen for code (Netlify/Vercel). You bring the code; they run the server.
  • SaaS: Finished software (Google Docs/Netflix). You just use it.
3. Deployment Models
  • Public: Shared hardware (AWS/Azure/GCP). Cheap and scalable.
  • Private: Dedicated hardware for one company. Expensive and high-security.
  • Hybrid: The "Best of Both." Sensitive data stays in Private; public site runs on Public.
4. Cloud Superpowers

Scalability is the ability to handle growth by adding power. Elasticity is the ability to scale back down automatically to save money. This "Pay-As-You-Go" model charges by the minute, preventing bankruptcy after a viral traffic spike.

Command Cheat-Sheet
git clone: Download a repo.
git pull: Get latest updates.
git push: Upload your code.
git commit: Save a local snapshot.
Test Alert!
Data Centers require massive cooling and backup generators to ensure "Five Nines" (99.999%) uptime during blackouts.
