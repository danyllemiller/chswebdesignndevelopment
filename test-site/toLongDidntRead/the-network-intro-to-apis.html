TL;DR: Chapter 15

API Communication, JSON & Async Mastery

The Network Rules

1. The API Concept

An API is the Waiter. The Client (User) places an order, the API takes it to the Kitchen (Server), and brings back the Response (Data).

2. Mapping CRUD to HTTP
  • GET (Read): View data.
  • POST (Create): Add new data.
  • PUT (Update): Change existing data.
  • DELETE (Delete): Remove data.
3. Handling the "Pause" (Async)

Network requests take time. Use async/await to pause the function without freezing the browser. fetch() returns a Promise (the pager), which eventually resolves into data.

4. JSON: The Language of Data

Data travels as a string. You must use response.json() (Deserialization) to turn it back into a usable JavaScript object.

5. Pro-Level Security

Never leak API Keys in client-side code. Use Rate Limiting to prevent abuse and try/catch to handle internet failures.

The Success Code
Always check response.ok. A 200 OK means victory; a 404 means the endpoint address is wrong.
Stateless Rule
REST APIs are "Stateless." The server forgets you after every request. You must provide all ID info every time.
