Tech Talk: Chapter 15

Exhaustive Vocabulary for APIs & Data Exchange

API (Application Programming Interface)
A set of rules and protocols that allows different software applications to communicate with each other.
Asynchronous
Programming that allows a process to run in the background while other tasks continue, preventing the UI from freezing.
Await
A keyword used inside async functions to pause execution until a Promise resolves.
CORS (Cross-Origin Resource Sharing)
A security mechanism that allows or restricts requested resources on a web page from another domain outside the domain from which the resource originated.
CRUD
Stands for Create, Read, Update, and Delete—the four basic functions of persistent storage.
Deserialization
The process of converting a JSON string received from a network back into a usable JavaScript object.
Endpoint
A specific URL where an API receives requests about a specific resource.
Fetch
The modern JavaScript method used to make network requests across the web.
JSON (JavaScript Object Notation)
A lightweight text-based data format that is easy for humans to read and for machines to parse.
Payload
The actual data transmitted in an HTTP request (the "body" of a POST or PUT request).
Promise
An object representing the eventual completion or failure of an asynchronous operation.
Rate Limiting
A strategy for limiting network traffic to prevent users from making too many requests in a short time.
REST (Representational State Transfer)
The industry-standard architectural style for building web APIs using HTTP methods.
Status Code
A 3-digit number returned by a server indicating the result of an HTTP request (e.g., 200, 404, 500).
