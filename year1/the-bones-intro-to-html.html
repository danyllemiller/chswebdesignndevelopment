What is an HTML File? (The Boilerplate)

Every HTML file starts with a friendly greeting called the document type declaration, or doctype. It's a special tag that tells the browser what kind of code it's about to read. For HTML5, the newest version, it's super simple: <!DOCTYPE html>.

You can think of every HTML document as a family tree with three main parts. This whole structure is called the "boilerplate."

  • <html>: The root element that holds everything together. A common and important attribute to add to this tag is lang (language), like <html lang="en">. This tells search engines and screen readers that the page is in English, which is important for accessibility and SEO.
  • <head>: The brain of the page, holding all the important information that the user doesn't see, like the page's title. Besides the <title> tag, the <head> is also where you put crucial meta tags. These are invisible tags that provide data about your page. Three of the most important ones you should always include are:
    • <meta charset="UTF-8">: This tells the browser to use the standard UTF-8 character set, which supports almost all characters and symbols.
    • <meta name="viewport" content="...">: This is essential for responsive design. It tells mobile phones to use their actual screen width.
    • <meta name="description" content="...">: This provides a short summary of your page for search engine results, like Google.
  • <body>: The main part of the page, holding all the content the user does see, like text, images, and videos.

Walkthrough: Create the Index Page (Initial Boilerplate)

This is the first page users see. All web servers look for a file named index.html to serve as a website's home page.

  1. Open a New File: Open a new text file in your code editor (like VS Code) and save it as index.html.
  2. Write the Structure: In VS Code, type ! and press Tab (this is an Emmet shortcut!). Watch the boilerplate magically appear. Update the title and metadata to match the example below: 
    <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="PixelPlay: Retro Arcade.">
  <title>PixelPlay: Your Ultimate Retro Arcade Destination</title>
</head>
<body>
  <p>Initial Boilerplate Content.</p>
</body>
</html>
  3. Test It: Save your file, double-click it in your computer's file explorer, and it will open in your web browser. You should see a blank page with your single line of text!

What is an HTML "Tag"?

HTML is all about using element tags to mark different parts of your page. Most tags come in pairs, with a starting tag and an ending tag. This creates an element.

  • Some elements, like a line break, are empty elements and only need one tag: <br>.
  • Tags can also be nested, which means you put one tag completely inside another. Think of it like putting one box inside a bigger box.

The "First-In, Last-Out" Rule

When you nest tags, remember this golden rule: the last tag you open is the first one you have to close. This concept is often called proper nesting.

Think of it like packing a suitcase. You put your clothes inside a packing cube, and then you put the packing cube inside the suitcase. You have to zip up the packing cube (the inner container) before you can zip up the suitcase (the outer container). If you try to close the suitcase while the packing cube is still half-open and sticking out, it won't work!

Correct Nesting:
<p>This is <em>correct</em> nesting.</p>
Incorrect Nesting (Overlapping Tags):
<p>This is <em>incorrect nesting.</p></em>

<p class="intro">: Here, class is the attribute, and intro is its value.

The "Big 5" Semantic Tags (The Page Layout)

Now it's time to build the "body" of our page! We can use special tags called semantic elements because their names tell you exactly what kind of content they contain. This is great for SEO and for screen readers (accessibility).

💼 The Agency Founder's Log: Selling SEO

When pitching a Carson City business, anyone can build a visual layout using generic <div> tags or drag-and-drop website builders. As an agency, you sell 'Semantic Architecture.' By using proper tags like <article> and <nav>, you are explicitly telling Google how to read the site, which boosts your client's search rankings. Higher rankings = more revenue for them, which means higher development fees for you!

The Blueprint: Why We Wireframe

Think of a wireframe as the architectural blueprint for your website. Just as you wouldn't build a house without a plan, you shouldn't build a website without a wireframe. Developers use them to map out exactly where the semantic areas (like the header, nav, and footer) will live before getting distracted by colors, fonts, or images.

Why are they so important? This step is critical because it forces you to think about the structure of your data. If you skip this step, you risk writing messy, unorganized markup that is hard to fix later. By planning with boxes and labels first, you ensure your HTML tags will be nested correctly and your content will be logical.

  • <header>: The introduction of a section or page.
  • <nav>: Marks a list of hypertext or navigation links. [HTML5]
  • <main>: Marks the main content of the document; only one main element should be used. [HTML5]
  • <article>: A self-contained piece of content, like a blog post or news story. [HTML5]
  • <footer>: The closing content of a section or page, often with contact info. [HTML5]
  • <h1> to <h6>: Headings that mark the most important text.

Other Sectioning & Grouping Elements

Grouping elements are essential for keeping your webpage organized and manageable. Think of them as the containers or boxes that hold related pieces of content together. By wrapping specific parts of your page in these elements, you can target them as a group for styling with CSS or applying behavior with JavaScript.

For example, you might group a headline, an image, and a paragraph together to create a "card" component that stands out from the rest of the page. Beyond just styling, these elements provide structure that helps browsers and screen readers understand the relationships between different parts of your content. While a <div> is a generic container with no specific meaning, elements like <section> and <article> add semantic value, telling the browser exactly what purpose that group of content serves.

Hypertext Links

What made the World Wide Web different from just reading a digital book? It's right in the name of the language: HyperText. Links are created with the <a> (anchor) tag. href is the address, and the content between the tags is the clickable part.

  • mailto: opens email (e.g., <a href="mailto:test@test.com">Email Us</a>).
  • tel: allows mobile users to tap to call.
  • target="_blank" forces the link to open in a brand new tab.
Relative vs. Absolute Paths

Absolute Path: The full URL to a completely different website (e.g., https://google.com).
Relative Path: The path from the current file you are editing to a target file on your own computer (e.g., about.html). If you break a relative path, the user gets a 404 Error!

Walkthrough: Link It All Together (Navigation)

  1. Build the Nav: Open your index.html. Inside the <header>, find the <nav><ul> you built earlier. Wrap the text of your list items in anchor tags (<a>) pointing to your other pages using relative paths.

    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="rentals.html">Rentals</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>
  2. Duplicate Across Pages: Copy this entire <nav> block and paste it into the <header> of all your other HTML files (about.html, contact.html, etc.) so the user can navigate anywhere from any page!

Accessible Forms & User Input

Without forms, the web is just a one-way TV broadcast. Forms allow the user to talk back! Think of Instagram—what HTML element do you type your comments into? An input field inside a form.

The <form> tag acts as a wrapper for all user inputs. Inside it, we use tags like <input type="text"> for short answers, <textarea> for long messages, and <button type="submit"> to send the data.

Cultural Competency: The Label Rule

A visually impaired user using a screen reader cannot see the box on the screen. If you just put a generic input box, the screen reader will just say "text field." To fix this, every input MUST have a matching <label>.

We link them together using the for attribute on the label, and the id attribute on the input. They must match perfectly!

Walkthrough: Add a Contact Form

  1. Open Contact Page: Open your contact.html file. Inside the <main> area, create a <form> block.
  2. Build the Inputs: Add a labeled input for Name, a labeled input for Email, and a labeled textarea for the Message. We will also use the HTML5 required attribute so the user can't submit a blank form! 
    <form>
    <label for="userName">Full Name:</label>
    <input type="text" id="userName" name="userName" required>
    
    <label for="userEmail">Email Address:</label>
    <input type="email" id="userEmail" name="userEmail" required>

    <label for="userMessage">Your Message:</label>
    <textarea id="userMessage" name="userMessage" rows="4" required></textarea>

    <button type="submit">Send Message</button>
</form>
