Version Control (VCS)

Professional teams don't email files. They use Git to track changes and GitHub to collaborate. This is your code "Time Machine."

The Command Field Guide:
git clone [url] Download a whole project
git commit -m "msg" Save a local snapshot
git pull Get updates from team
git push Upload your work
Branching Strategy:
  • Main / Master: The definitive, working version of your site.
  • Feature Branch: Isolated copy for testing new ideas.
  • Merge Conflict: When two people edit the same line. You must resolve it manually!

Service Models

Choosing how much you want to manage.

IaaS (Infrastructure)

"Take & Bake": Provider gives raw hardware. You install the OS and the software.

PaaS (Platform)

"Pizza Delivery": Provider manages the server/OS. You just bring the code (Netlify).

SaaS (Software)

"Dine-In": Provider manages everything. You just use it (Netflix / Google Docs).

🏭

Infrastructure Anatomy

"The Cloud" is actually physical computers in a warehouse.

  • Servers: Powerful computers running an Operating System (OS) like Linux.
  • Cooling: Massive industrial AC units prevent hardware from melting under load.
  • Redundancy: Giant backup generators ensure the site stays online during blackouts.
  • CDN: A global network that stores your media closer to the user to reduce lag.
☁️

Deployment Models

Where your data physically lives.

PUBLIC CLOUD

Shared resources (AWS). Like renting an apartment in a skyscraper. Cheap & Scalable.

PRIVATE CLOUD

Dedicated resources for one organization. Like owning a home on a private island. Secure & Expensive.

HYBRID CLOUD

Best of both. Sensitive data in Private; Public-facing site in Public.

💪

Cloud Superpowers

How big sites handle traffic spikes.

Scalability:

The ability to grow and add power when traffic increases (e.g., from 100 to 50,000 users).

Elasticity:

The ability to automatically shrink back down when traffic drops to save money.

Business Model: Pay-as-you-go (by the minute/second).
