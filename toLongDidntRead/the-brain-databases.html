TL;DR: Chapter 16

Exhaustive Database & SQL Mastery Summary

The Core Concepts

1. SQL vs. NoSQL vs. Local
  • SQL (Relational): Strict tables/rows/columns. Uses Primary Keys (unique IDs) and Foreign Keys (links) to normalize data. Best for banking and complex relations.
  • NoSQL (Non-Relational): Flexible JSON-like Documents stored in Collections. Best for real-time data like chat apps.
  • Local Storage: Browser-only memory (5MB limit). Saves data as Strings. Use JSON.stringify() to save objects and JSON.parse() to read them.
2. Mastering CRUD & SQL Commands
  • Create: INSERT INTO table_name (cols) VALUES (vals);
  • Read: SELECT cols FROM table WHERE condition ORDER BY col DESC LIMIT 10;
  • Update: UPDATE table SET col=val WHERE condition;
  • Delete: DELETE FROM table WHERE condition; (Removing structure entirely is DROP TABLE).
  • Relate: JOIN combines tables based on related columns. LIKE 'Dan%' uses the % wildcard for partial matches.
3. Professional Security Protocols
  • SQL Injection (SQLi): Malicious code typed into inputs. Fix: Use Parameterized Queries to sanitize input.
  • Hashing: One-way math "meat-grinder" for passwords. Professionals use bcrypt or Argon2.
  • Salting: Adding random noise to passwords before hashing to stop Rainbow Table attacks.
  • Roles: Never give a web app Root/Admin access; if hacked, the database is fully exposed.
The Golden Rule
NEVER run an UPDATE or DELETE without a WHERE clause. Without it, you will modify or wipe the entire table!
The Pro Toolbox
ORM: Lets you talk to a DB using JavaScript instead of raw SQL.
Index: Speeds up searches like a book's index.
Persistence Rule
Local Storage is for preferences (Dark Mode). Server Databases are for critical data (Scores/Accounts).
