TL;DR: Chapter 11

JavaScript Logic & DOM Manipulation Cheat Sheet

The Core "Rules" of Scripting

1. Declare Variables Correctly
  • const: Use for HTML elements and values that never change. (Prevents TypeError).
  • let: Use for counters, totals, or data that fluctuates.
  • Scope: Keep variables Local (inside functions) whenever possible to prevent logic conflicts.
2. Strict Equality is Mandatory

Never use ==. Always use Strict Equality (===) to check both the value and the data type. This prevents the browser from "guessing" your logic.

3. The Action Funnel

Every interaction follows this path: Listen (Event) → Process (Function) → Update (DOM Manipulation).

  • document.querySelector() is your remote control.
  • addEventListener() is your watchman.
4. Dynamic Data Rendering

Real sites use Loops to build HTML from Arrays of Objects. Follow the 3-step engine: Create (createElement) → Fill (innerHTML/Backticks) → Attach (appendChild).

Manager's Golden Rule
Always use event.preventDefault() inside form submit listeners. This intercepts the data on the Client-Side before the browser refreshes the page.
Debugging Secret
If text isn't appearing, check the Console. A ReferenceError means you used a variable that doesn't exist. undefined means you used the wrong property name (e.g., pet.Name instead of pet.name).
